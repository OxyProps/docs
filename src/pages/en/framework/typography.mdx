## Font Families

The font-family property in CSS allows you to specify the font that will be used for text content on your website. OxyProps provides a range of font families that can be easily used in your CSS styles or with utility classes.

To use a font family with CSS, you can set the font-family property to the name of the font you want to use, like this:

```css
body {
  font-family: "Open Sans", sans-serif;
}
```

In this example, we're setting the font-family of the body element to "Open Sans", which is a popular sans-serif font. The sans-serif fallback ensures that if "Open Sans" is not available on the user's device, a generic sans-serif font will be used instead.

### OxyProps Framework

OxyProps includes four font families available as CSS variables:

```css
:where(html) {
  --o-font-sans: system-ui, -apple-system, Segoe UI, Roboto, Ubuntu, Cantarell,
    Noto Sans, sans-serif, Segoe UI, Roboto, Ubuntu, Cantarell, Noto Sans,
    sans-serif, Segoe UI, Roboto, Ubuntu, Cantarell, Noto Sans, sans-serif;
  --o-font-serif: ui-serif, serif;
  --o-font-mono: Dank Mono, Operator Mono, Inconsolata, Fira Mono, ui-monospace,
    SF Mono, Monaco, Droid Sans Mono, Source Code Pro, monospace;
}
```

<ClipboardCopy value="var(--o-font-sans)"><code>--o-font-sans</code></ClipboardCopy>: This family stack includes popular sans-serif fonts such as Roboto, Ubuntu, and Noto Sans. It also includes system fonts like Segoe UI and -apple-system, making it a versatile option for UI design.

`--o-font-serif`: This family stack includes ui-serif, a system font commonly used for serif text, as well as other serif options.

`--o-font-mono`: This family stack includes popular monospace fonts like Inconsolata and Fira Mono, as well as system fonts like SF Mono and Droid Sans Mono.

These 3 stacks use system fonts. This means that the fonts will be automatically updated when the user updates their operating system. This is a great way to ensure that your website will always look great on the latest operating systems.

Using system fonts instead of external fonts from services like Google Fonts can have several benefits:

1. **Faster Page Load**: Since system fonts are already installed on the user's device, they don't need to be downloaded, which can significantly reduce page load times.
2. **Consistency**: System fonts are consistent across different devices and platforms. This means that the fonts on your website will look the same regardless of the device or platform that the user is using. In contrast, external fonts may not be available on all devices, and their appearance may vary depending on the device's settings.
3. **Privacy**: External fonts are loaded from third-party servers, which means that the user's data is being sent to those servers. Using system fonts eliminates this potential privacy concern.
4. **Design integrity**: System fonts are designed to work well in various scenarios, such as different font sizes, weights, and styles. This is because system fonts are optimized for the operating system they are used on. On the other hand, external fonts can sometimes appear distorted or blurry when used in certain scenarios.

Overall, using system fonts can result in a faster, more consistent, and more private user experience while maintaining the design integrity of your website.

`--o-font-atkinson`: This family stack includes "Atkinson Hyperlegible", a font named after Braille Institute founder, J. Robert Atkinson. Atkinson Hyperlegible focuses on letterform distinction to increase character recognition and improve readability.

This one is a self hosted font. Using self-hosted fonts means that the font files are stored on the same server as your website or application, rather than being loaded from a third-party service like Google Fonts. There are several benefits to using self-hosted fonts instead of external fonts:

1. **Improved page load speed**: By hosting the font files on your own server, you can reduce the number of external requests your website needs to make, which can help to improve page load times.
2. **Increased control**: When you self-host your fonts, you have complete control over the font files and how they are served. This means that you can optimize the font files for your specific use case and ensure that they are served in the most efficient way possible.
3. **Better privacy and security**: By not relying on third-party services to serve your fonts, you can help to protect your users' privacy and security. When you use external font services, you are effectively outsourcing a part of your website to a third party, which could potentially compromise your users' data.

To use these font families in your project, you can use the following utility classes:

`.o-sans`: Sets the font family to `var(--o-font-sans)`  
`.o-serif`: Sets the font family to `var(--o-font-serif)`  
`.o-mono`: Sets the font family to `var(--o-font-mono)`  
`.o-atkinson`: Sets the font family to `var(--o-font-atkinson)`  

To apply any of these classes, simply add the appropriate class to your HTML element. For example, to set the font family to `--o-font-sans`, add the class `o-sans` to your element:

```html
<p class="o-sans">This text will use the sans-serif font family.</p>
```
