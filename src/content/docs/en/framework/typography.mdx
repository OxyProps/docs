## Font Families

The font-family property in CSS is a powerful tool that allows web designers and developers to specify the font used for text content on a website. By setting the font-family property to a specific font, designers can control the visual appearance of text content and create a consistent visual identity for a website.

Choosing the right font family is crucial for creating a website that is both aesthetically pleasing and functional. Different font families can convey different moods and emotions, and some are more legible than others at different sizes and in different contexts.

In this section, we will explore the font families available in OxyProps, which can be easily used in your CSS styles or with utility classes. We will discuss the benefits of using system fonts and self-hosted fonts, and provide examples of how to use the OxyProps font families in your web projects.

### OxyProps font families

Oxyprops offers 4 font families available as CSS variables. 3 of them are using your website visitor system fonts instead of external server-hosted fonts. This has several benefits.  

Firstly, system fonts are pre-installed on most devices, which means that the website's text will load faster as the browser doesn't have to download the font file from an external server. This can lead to improved website performance and user experience.  

Additionally, system fonts are typically designed to be highly readable and legible, making them a reliable choice for website text content.  

By using Oxyprops font families, which are system fonts optimized for the web, developers can ensure a consistent and optimized text experience across various devices and platforms.

The font stack `system-ui, -apple-system, Segoe UI, Roboto, Ubuntu, Cantarell, Noto Sans, sans-serif` is interesting to use because it specifies a range of font families that are commonly available on different operating systems and devices.

The `system-ui` and `-apple-system` values are generic font family names that will use the default UI font of the operating system or device, which ensures that the website text will match the overall look and feel of the device or operating system.

Using a font stack that includes popular monospace fonts like `Dank Mono`, `Inconsolata`, and `Source Code Pro` can improve the readability and legibility of text content on a website or application. Monospace fonts are designed with fixed-width characters that align horizontally, which makes them an ideal choice for coding, terminal, and other technical contexts where visual clarity is critical. Including fallbacks like `ui-monospace` and `monospace` ensures that the text will display legibly on a wider range of devices and browsers.

:::tip[exact character count with ch]
The `ch` unit is a relative unit that is based on the width of the character "0" in the font that is specified. In non-monospace fonts, using `ch` can result in slightly varying widths, as characters may have different widths depending on their shape. However, when using a monospace font, each character has an equal width, so the `ch` unit will always be equivalent to the width of a single character. For this reason, OxyProps uses `ch` as the default unit for text width. When using the `--o-font-mono` family, this ensures that `60ch` will always be exactly 60 characters wide, regardless of the specific characters used.
:::

## Letter Spacing

Letter spacing, also known as tracking, refers to the amount of space between each character in a word. It can be adjusted to create a specific look or improve the readability of the text. The letter spacing can be increased or decreased depending on the desired effect.

OxyProps includes a range of CSS variables and utility classes for controlling letter spacing. The available CSS variables and utility classes range from -0.5em to 1em, with the values increasing. For example, the o-tracking-1 class adds a letter spacing of 0.025em to the text, while the o-tracking-7 class adds a letter spacing of 1em.

## Font Sizes

Font size is a crucial aspect of typography. It determines the size of the characters in relation to the rest of the text and the overall layout. In CSS, font size is typically set in pixels (px, which I don’t recommend), ems (em), or rems (rem).

In the OxyProps typography system, font sizes are defined using rem units. Rem units are similar to em units but are relative to the root element, making them more consistent across different elements and devices.

The default font size in the OxyProps system is 1rem, which is equivalent to the default font size of the browser (usually 16px). From there, the system provides several predefined font sizes that can be easily applied to elements using CSS classes or variables.

### Fluid Font Sizes

Fluid font sizes are responsive font sizes that change based on the size of the screen. They are defined using the CSS clamp() function, which allows to define a minimum, maximum, and preferred font size. The preferred font size is the font size that will be used when the screen size is between the minimum and maximum values.

### Advanced Fluid Font Sizes

This set of fluid font sizes is defined using the CSS variables below. These variables are based on the fluid font sizes defined in the OxyProps system, but they can be modified to create custom fluid font sizes by adjusting the Typography parameter in the plugin dashboard.

You can customize the fluid font sizes system by adjusting 5 parameters:

- **Base font size:** The base font size for your website.
- **Font scale ratio:** The scale ratio used to calculate the fluid font sizes.
- **Mobile to Desktop ratio:** The ratio between font sizes at smallets breakpoint and largest breakpoint.
- **Absolute minimum size:** The minimum font size whatever the screen size.
- **Heading to subheading ratio:** The ratio between a heading and its subheading font size.

#### Base font size

This settings defines the base font size for your website. It will define the <ClipboardCopy value="var(--o-fluid-font-0)">`--o-fluid-font-0`</ClipboardCopy> value and will be used as the default for your body text and your `<h6>` headings. Its value is given in rems.

:::note[desktop is the base value]
As most of us develop websites on a desktop computer, the base font size is defined for desktop at maximum viewport width. This means that the base font size will be smaller on mobile devices. See [Mobile to Desktop ratio](#mobile-to-desktop-ratio) for more information.
:::

Define your base font size either by entering a value in the input field or by using the slider. In the following example, the base font size is set to 1.2rems. On most browsers with the default font size set to 16px, this will result in a font size of 19.2px. But is a user has set their browser font size to 20px, the base font size will be 24px for them.

<ContentImage coverImage="/images/oxyprops-typography-base-size.webp" alt="Screenshot of the OxyProps base font size typography setting." aspectRatio="16:9" />

:::tip[Why use rems?]
It is important for accessibility to define font sizes in rems because it allows users to adjust the font size according to their needs without compromising the layout or functionality of the website.  

When font sizes are defined in pixels, they are fixed and do not respond to the user’s browser settings. This can make the text too small or too large for some users, especially those with visual impairments or who use assistive technology.  

On the other hand, when font sizes are defined in rems, they are relative to the root element, which is usually set to the default font size of the browser. This means that when the user adjusts the font size in their browser settings, all font sizes defined in rems will adjust accordingly, maintaining the relative size and layout of the website.  

This is important for accessibility because it ensures that all users, regardless of their visual abilities or preferences, can read and interact with the content on the website without difficulty.  
:::

#### Font scale ratio

This setting defines the scale ratio used to calculate all the fluid font sizes. Starting from the base font size, which is <ClipboardCopy value="var(--o-fluid-font-0)">`--o-fluid-font-0`</ClipboardCopy>, it is multiplied by the scale ratio to get the next font size. This process is repeated until the maximum font size is reached. This reasults in an exponential scale and not a linear scale, which gives a better visual hierarchy.

As an example, if you define your base font size to be 1rem and your scale ratio to be 1.25, the fluid font sizes will be calculated as follows:

- <ClipboardCopy value="var(--o-fluid-font-00)">`--o-fluid-font-00`</ClipboardCopy> = 1rem ÷ 1.25 = 0.8rem
- <ClipboardCopy value="var(--o-fluid-font-0)">`--o-fluid-font-0`</ClipboardCopy> = 1rem
- <ClipboardCopy value="var(--o-fluid-font-1)">`--o-fluid-font-1`</ClipboardCopy> = 1rem × 1.25 = 1.25rem
- <ClipboardCopy value="var(--o-fluid-font-2)">`--o-fluid-font-2`</ClipboardCopy> = 1rem × 1.25 × 1.25 = 1.5625rem
- <ClipboardCopy value="var(--o-fluid-font-3)">`--o-fluid-font-3`</ClipboardCopy> = 1rem × 1.25 × 1.25 × 1.25 = 1.953125rem
- <ClipboardCopy value="var(--o-fluid-font-4)">`--o-fluid-font-4`</ClipboardCopy> = 1rem × 1.25 × 1.25 × 1.25 × 1.25 = 2.44140625rem
- <ClipboardCopy value="var(--o-fluid-font-5)">`--o-fluid-font-5`</ClipboardCopy> = 1rem × 1.25 × 1.25 × 1.25 × 1.25 × 1.25 = 3.0517578125rem
- etc.

In the following example, the font scale ratio is set to 1.33 which corresponds to the P4 (Perfect fourth) preset.

Presets are defined after the musical scale, which is a series of notes that increase in frequency by a factor of 2. The P4 preset is the fourth note of the scale, which is a 33% increase in frequency.

#### Mobile to Desktop ratio

This setting defines the ratio between font sizes at smallest breakpoint and largest breakpoint. This is useful when you want to have a different font size on mobile devices than on desktop devices. For example, you might want to have a smaller font size on mobile devices to make it easier to read.

If you set this value to 1, the font size will be the same on all devices. If you set it to 2, the font size will be half the size on mobile devices. Or in other words, the font size will be doubled on desktop devices.

In the following example, the mobile to desktop ratio is set to 1.78 which corresponds to the m7 (minor seventh) preset.

With this preset, if my desktop base font size is set to 1.2rem, the mobile base font size will be 1.2rem ÷ 1.78 = 0.67rem.

## Text Orientation

Text orientation refers to the direction of text flow and the orientation of individual characters within the text. By default, text flows horizontally from left to right in the Western world. However, in some cases, it may be necessary to display text vertically, either top to bottom or bottom to top. The o-text-down and o-text-up utility classes are used to set the text flow direction to top to bottom and bottom to top, respectively.

The o-upright class is used to set the orientation of individual characters within the text to upright, meaning that each character is displayed in its standard horizontal orientation, even if the text is displayed vertically. This is particularly useful when displaying text in East Asian languages, such as Chinese or Japanese, which have characters that are traditionally displayed in a vertical orientation. But it is amso very useful for design considerations, such as when displaying text vertically in a narrow column.

## Text Alignment

Text alignment refers to the horizontal positioning of text within a container. The different values for text alignment determine where the text is placed in relation to the edges of the container.

- **left**: The text is aligned with the left edge of the container.
- **right**: The text is aligned with the right edge of the container.
- **center**: The text is centered horizontally within the container.
- **justify**: The text is spread out to fill the width of the container, with extra space added between words as necessary.
- **start**: The text is aligned to the start edge of the container, which is typically the left edge in left-to-right languages and the right edge in right-to-left languages.
- **end**: The text is aligned to the end edge of the container, which is typically the right edge in left-to-right languages and the left edge in right-to-left languages.

## Text Overflow

Text overflow occurs when the content of a text container exceeds its specified width or height, and there is not enough space to display all of the content. When this happens, the text can be clipped or truncated, or an ellipsis (...) can be added to indicate that there is more content that is not currently visible.

Additionally, you can use the "line-clamp" property to limit the number of lines of text that are displayed, which can be useful for creating a more compact and visually appealing layout.

The text overflow utility classes can help you easily apply these effects to your text containers in a consistent and reusable way.

> **Note:**
