## Font Families

The font-family property in CSS allows you to specify the font that will be used for text content on your website. OxyProps provides a range of font families that can be easily used in your CSS styles or with utility classes.

<ContentImage coverImage="/images/oxyprops-font-families.webp" alt="Preview of the OxyProps font families in Bricks editor." aspectRatio="16:9" />

To use a font family with CSS, you can set the font-family property to the name of the font you want to use, like this:

```css
body {
	font-family: 'Open Sans', sans-serif;
}
```

In this example, weâ€™re setting the font-family of the body element to "Open Sans", which is a popular sans-serif font. The sans-serif fallback ensures that if "Open Sans" is not available on the userâ€™s device, a generic sans-serif font will be used instead.

OxyProps includes four font families available as CSS variables:

```css
:where(html) {
	--o-font-sans: system-ui, -apple-system, Segoe UI, Roboto, Ubuntu, Cantarell, Noto Sans, sans-serif, Segoe UI, Roboto, Ubuntu, Cantarell, Noto Sans, sans-serif, Segoe UI, Roboto,
		Ubuntu, Cantarell, Noto Sans, sans-serif;
	--o-font-serif: ui-serif, serif;
	--o-font-mono: Dank Mono, Operator Mono, Inconsolata, Fira Mono, ui-monospace, SF Mono, Monaco, Droid Sans Mono, Source Code Pro, monospace;
}
```

These 3 stacks use system fonts. This means that the fonts will be automatically updated when the user updates their operating system. This is a great way to ensure that your website will always look great on the latest operating systems.

Using system fonts instead of external fonts from services like Google Fonts can have several benefits:

1. **Faster Page Load**: Since system fonts are already installed on the userâ€™s device, they donâ€™t need to be downloaded, which can significantly reduce page load times.
2. **Consistency**: System fonts are consistent across different devices and platforms. This means that the fonts on your website will look the same regardless of the device or platform that the user is using. In contrast, external fonts may not be available on all devices, and their appearance may vary depending on the deviceâ€™s settings.
3. **Privacy**: External fonts are loaded from third-party servers, which means that the userâ€™s data is being sent to those servers. Using system fonts eliminates this potential privacy concern.
4. **Design integrity**: System fonts are designed to work well in various scenarios, such as different font sizes, weights, and styles. This is because system fonts are optimized for the operating system they are used on. On the other hand, external fonts can sometimes appear distorted or blurry when used in certain scenarios.

Overall, using system fonts can result in a faster, more consistent, and more private user experience while maintaining the design integrity of your website.

1. **Improved page load speed**: By hosting the font files on your own server, you can reduce the number of external requests your website needs to make, which can help to improve page load times.

## Letter Spacing

Letter spacing, also known as tracking, refers to the amount of space between each character in a word. It can be adjusted to create a specific look or improve the readability of the text. The letter spacing can be increased or decreased depending on the desired effect.

OxyProps includes a range of CSS variables and utility classes for controlling letter spacing. The available CSS variables and utility classes range from -0.5em to 1em, with the values increasing. For example, the o-tracking-1 class adds a letter spacing of 0.025em to the text, while the o-tracking-7 class adds a letter spacing of 1em.

## Font Sizes

Font size is a crucial aspect of typography. It determines the size of the characters in relation to the rest of the text and the overall layout. In CSS, font size is typically set in pixels (px, which I donâ€™t recommend), ems (em), or rems (rem).

In the OxyProps typography system, font sizes are defined using rem units. Rem units are similar to em units but are relative to the root element, making them more consistent across different elements and devices.

The default font size in the OxyProps system is 1rem, which is equivalent to the default font size of the browser (usually 16px). From there, the system provides several predefined font sizes that can be easily applied to elements using CSS classes or variables.

### Fluid Font Sizes

Fluid font sizes are responsive font sizes that change based on the size of the screen. The preferred font size is the font size that will be used when the screen size is between the minimum and maximum values.

### Advanced Fluid Font Sizes

This set of fluid font sizes is defined using the CSS variables below. These variables are based on the fluid font sizes defined in the OxyProps system, but they can be modified to create custom fluid font sizes by adjusting the Typography parameter in the plugin dashboard.

You can customize the fluid font sizes system by adjusting 5 parameters:

- **Base font size:** The base font size for your website.
- **Font scale ratio:** The scale ratio used to calculate the fluid font sizes.
- **Mobile to Desktop ratio:** The ratio between font sizes at smallets breakpoint and largest breakpoint.
- **Absolute minimum size:** The minimum font size whatever the screen size.
- **Heading to subheading ratio:** The ratio between a heading and its subheading font size.

#### Base font size

This settings defines the base font size for your website. It will define the <ClipboardCopy value="var(--o-fluid-font-0)">`--o-fluid-font-0`</ClipboardCopy> value and will be used as the default for your body text and your `<h6>` headings. Its value is given in rems.

> ðŸ’¡ **Note:**
>
> As most of us develop websites on a desktop computer, the base font size is defined for desktop at maximum viewport width. This means that the base font size will be smaller on mobile devices.

Define your base font size either by entering a value in the input field or by using the slider. In the following example, the base font size is set to 1.2rems. On most browsers with the default font size set to 16px, this will result in a font size of 19.2px. But is a user has set their browser font size to 20px, the base font size will be 24px for them.

> ðŸ’¡ **Note:**
>
> It is important for accessibility to define font sizes in rems because it allows users to adjust the font size according to their needs without compromising the layout or functionality of the website.
>
> When font sizes are defined in pixels, they are fixed and do not respond to the userâ€™s browser settings. This can make the text too small or too large for some users, especially those with visual impairments or who use assistive technology.
>
> On the other hand, when font sizes are defined in rems, they are relative to the root element, which is usually set to the default font size of the browser. This means that when the user adjusts the font size in their browser settings, all font sizes defined in rems will adjust accordingly, maintaining the relative size and layout of the website.
>
> This is important for accessibility because it ensures that all users, regardless of their visual abilities or preferences, can read and interact with the content on the website without difficulty.

#### Mobile to Desktop ratio

This setting defines the ratio between font sizes at smallets breakpoint and largest breakpoint. This is useful when you want to have a different font size on mobile devices than on desktop devices. For example, you might want to have a smaller font size on mobile devices to make it easier to read.

If you set this value to 1, the font size will be the same on all devices. If you set it to 2, the font size will be half the size on mobile devices. Or in other words, the font size will be doubled on desktop devices.

This is particularly useful when displaying text in East Asian languages, such as Chinese or Japanese, which have characters that are traditionally displayed in a vertical orientation. ## Text Alignment

Text alignment refers to the horizontal positioning of text within a container. The different values for text alignment determine where the text is placed in relation to the edges of the container.

- **left**: The text is aligned with the left edge of the container.
- **right**: The text is aligned with the right edge of the container.
- **center**: The text is centered horizontally within the container.
- **justify**: The text is spread out to fill the width of the container, with extra space added between words as necessary.
- **start**: The text is aligned to the start edge of the container, which is typically the left edge in left-to-right languages and the right edge in right-to-left languages.
- **end**: The text is aligned to the end edge of the container, which is typically the right edge in left-to-right languages and the left edge in right-to-left languages.
